📝 HTML Форми

Створення інтерактивних та доступних веб-форм

🎯 Основи HTML форм

HTML форми - це основний спосіб збору інформації від користувачів на веб-сайтах:

  • <form> - контейнер для всіх елементів форми
  • <input> - основний елемент для введення даних
  • <label> - підпис для поля вводу (важливо для доступності)
  • <fieldset> - групування пов'язаних полів
  • <legend> - заголовок для fieldset
  • <button> - кнопка для відправки або скидання форми
/* Базова структура HTML форми */ <form action="/submit" method="POST"> <fieldset> <legend>Особиста інформація</legend> <label for="name">Ім'я:</label> <input type="text" id="name" name="name" required> <label for="email">Email:</label> <input type="email" id="email" name="email" required> <button type="submit">Відправити</button> </fieldset> </form>

🔤 Типи Input полів

HTML5 надає багато різних типів полів для різних видів даних. Кожен тип має свої особливості та можливості.

В окремій статті розглянуті всі 20+ типів полів: text, email, password, number, date, color, file та інші, з інтерактивними демо та прикладами використання.

✅ Валідація форм

HTML5 та JavaScript надають потужні можливості для перевірки коректності введених даних.

В окремій статті розглянуті HTML5 атрибути валідації, JavaScript Constraint Validation API, real-time валідація, UX кращі практики та готові рішення.

📱 Адаптивні форми

Оптимізація форм для мобільних пристроїв:

/* Адаптивні стилі для форм */ /* Базові responsive стилі */ .form-input { width: 100%; padding: 0.75rem; font-size: 16px; /* Запобігає zoom на iOS */ border-radius: 6px; border: 2px solid #e2e8f0; } /* Мобільні пристрої */ @media (max-width: 768px) { .form-row { grid-template-columns: 1fr; /* Одна колонка */ } .form-input { padding: 1rem; /* Більші кнопки для пальців */ font-size: 16px; /* Мінімум 16px для iOS */ } .form-button { width: 100%; padding: 1rem; font-size: 1.1rem; } } /* Оптимізація для сенсорних екранів */ .form-input:focus { transform: scale(1.02); /* Легке збільшення */ box-shadow: 0 0 0 3px rgba(102, 126, 234, 0.2); } /* Кращі кнопки для мобільних */ .form-button { min-height: 44px; /* Мінімум для комфортного торкання */ touch-action: manipulation; /* Відключає double-tap zoom */ }
📱 Мобільні поради:
  • inputmode: Вказує тип клавіатури (numeric, tel, email)
  • autocomplete: Допомагає браузерам автозаповнювати поля
  • Розмір 16px+: Запобігає zoom на iPhone
  • Великі кнопки: Мінімум 44x44px для пальців
  • Fieldset: Групує пов'язані поля візуально

♿ Доступність та UX

✅ Кращі практики доступності:
  • Label для кожного input: Зв'язуйте через for/id
  • Placeholder ≠ Label: Placeholder зникає при вводі
  • Зрозумілі помилки: Чіткі інструкції як виправити
  • Keyboard navigation: Tab порядок має бути логічним
  • Focus видимість: Чіткий індикатор фокуса
  • Aria-labels: Додаткові описи для screen readers
⚠️ Поширені помилки:
  • Відсутність labels для input полів
  • Placeholder як єдиний опис поля
  • Надто маленькі кнопки (<44px)
  • Незрозумілі повідомлення про помилки
  • Відсутність індикаторів обов'язкових полів
  • Погана контрастність кольорів
