🔤 HTML Input Types

Всі типи полів введення та їх можливості

🎯 Вступ до Input Types

HTML5 надає багато різних типів полів для різних видів даних. Кожен тип має свої особливості, валідацію та поведінку:

  • Автоматична валідація - браузер перевіряє коректність даних
  • Спеціальні клавіатури - на мобільних з'являється відповідна клавіатура
  • Вбудовані віджети - календарі, колірні палітри, слайдери
  • Покращена доступність - screen readers розуміють тип поля
  • Автозаповнення - браузери можуть підказувати значення
/* Базовий синтаксис input елемента */ <input type="тип" id="унікальний_id" name="назва_для_форми" placeholder="підказка" required> /* Приклад з усіма атрибутами */ <label for="email">Email адреса:</label> <input type="email" id="email" name="user_email" placeholder="user@example.com" required autocomplete="email">

📝 Текстові поля

text
<input type="text">
Звичайне текстове поле для коротких рядків тексту
Корисні атрибути:
  • maxlength - максимальна кількість символів
  • minlength - мінімальна кількість символів
  • pattern - регулярний вираз для валідації
  • autocomplete - підказки браузера
password
<input type="password">
Приховує введений текст, автозаповнення паролів
Особливості:
  • Символи відображаються як крапки/зірочки
  • Браузери пропонують зберегти пароль
  • Можна використовувати з менеджерами паролів
  • Підтримує autocomplete="current-password"
search
<input type="search">
Поле пошуку з кнопкою очищення
Переваги:
  • Автоматична кнопка "X" для очищення
  • Історія пошукових запитів
  • Оптимізація для пошукових алгоритмів
  • Кращі стилі в деяких браузерах
textarea
<textarea rows="5"></textarea>
Багаторядкове текстове поле
Атрибути:
  • rows - кількість рядків
  • cols - кількість колонок
  • resize - CSS для контролю зміни розміру
  • wrap - переніс тексту

📧 Поля з валідацією

email
<input type="email">
Автоматична валідація email адреси, спеціальна клавіатура на мобільних
Можливості:
  • Перевірка формату email (містить @)
  • Клавіатура з @ та .com на мобільних
  • Підтримує multiple для кількох email
  • Автозаповнення з адресної книги
url
<input type="url">
Автоматична валідація URL адрес
Валідація:
  • Перевіряє наявність протоколу (http/https)
  • Валідує структуру URL
  • Клавіатура з .com/.org на мобільних
  • Підказки з історії браузера
tel
<input type="tel">
Телефонні номери, спеціальна клавіатура на мобільних
Особливості:
  • Цифрова клавіатура на мобільних
  • Без автоматичної валідації (формати різні)
  • Використовуйте pattern для валідації
  • Підтримує autocomplete="tel"

🔢 Числові поля

number
<input type="number" min="1" max="100">
Тільки числа, кнопки +/-, атрибути min/max
Атрибути:
  • min - мінімальне значення
  • max - максимальне значення
  • step - крок зміни (0.1, 5, any)
  • placeholder - підказка
range
<input type="range" min="0" max="100">
Значення: 50
Слайдер для вибору значення в діапазоні
Використання:
  • Гучність, яскравість, швидкість
  • Оцінки, рейтинги
  • Прогрес-бари
  • Налаштування фільтрів

📅 Дата та час

date
<input type="date">
Вбудований календар для вибору дати
Формат:
  • Значення: YYYY-MM-DD (2024-03-15)
  • Відображення залежить від локалі
  • Підтримує min/max для діапазону
  • step для кроку в днях
time
<input type="time">
Вибір часу з годинами та хвилинами
Формат:
  • Значення: HH:MM або HH:MM:SS
  • 24-годинний формат в коді
  • Відображення залежить від системи
  • step для точності (секунди)
datetime-local
<input type="datetime-local">
Дата та час без часового поясу
Використання:
  • Бронювання, зустрічі
  • Дедлайни, нагадування
  • Формат: YYYY-MM-DDTHH:MM
  • Без часового поясу
month
<input type="month">
Вибір місяця та року
Приклади:
  • Звіти по місяцях
  • Термін дії карток
  • Формат: YYYY-MM
  • Зручно для періодів
week
<input type="week">
Вибір тижня в році
Формат:
  • YYYY-W##
  • W01 - перший тиждень року
  • Планування по тижнях
  • Розклади, графіки

🎨 Спеціальні типи

color
<input type="color" value="#667eea">
Колірна палітра для вибору кольору
Можливості:
  • Значення у форматі #RRGGBB
  • Вбудований color picker
  • Підтримує тільки hex кольори
  • Не підтримує прозорість
file
<input type="file" accept=".jpg,.png">
Завантаження файлів з фільтром по типу
Атрибути:
  • accept - типи файлів (.jpg, image/*, .pdf)
  • multiple - кілька файлів
  • capture - камера на мобільних
  • Потребує enctype="multipart/form-data"
hidden
<input type="hidden" value="data">
Поле не відображається
(передає дані на сервер)
Приховане поле для передачі даних
Використання:
  • ID записів для редагування
  • CSRF токени безпеки
  • Версії для оптимістичного блокування
  • Стан форми

☑️ Чекбокси та радіо

checkbox
<input type="checkbox" checked>
Множинний вибір, так/ні варіанти
Особливості:
  • Може бути checked або unchecked
  • Значення передається тільки якщо checked
  • Група з однаковим name[] для масиву
  • indeterminate стан через JavaScript
radio
<input type="radio" name="group">
Вибір одного варіанту з групи
Правила:
  • Однаковий name для групи
  • Тільки один може бути вибраний
  • Різні value для кожного варіанту
  • Завжди групувати в fieldset

🔘 Кнопки форм

submit
<input type="submit" value="Відправити">
Відправляє форму на сервер
Поведінка:
  • Запускає валідацію форми
  • Відправляє дані на action URL
  • Можна перевизначити formaction
  • Краще використовувати <button type="submit">
reset
<input type="reset" value="Скинути">
Скидає всі поля до початкових значень
Рідко використовується:
  • Погана UX - користувачі втрачають дані
  • Не запитує підтвердження
  • Краще зробити кастомну кнопку
  • Або не використовувати взагалі
button
<input type="button" value="Дія">
Кнопка без дії за замовчуванням
Використання:
  • JavaScript обробники подій
  • AJAX запити
  • Динамічна поведінка
  • <button></button> більш гнучкий
image
<input type="image" src="button.png">
Графічна кнопка відправки
(рідко використовується)
Графічна кнопка для відправки форми
Особливості:
  • Працює як submit кнопка
  • Передає координати кліка (x, y)
  • Потребує src та alt атрибути
  • Краще використовувати CSS кнопки

📋 Списки та вибір

Select елементи

Простий select

<select name="country"> <option value="">Оберіть країну</option> <option value="ua">Україна</option> <option value="us">США</option> </select>

Групований select

<select name="tech"> <optgroup label="Frontend"> <option value="html">HTML</option> <option value="css">CSS</option> </optgroup> <optgroup label="Backend"> <option value="php">PHP</option> </optgroup> </select>

Множинний вибір

<select name="skills[]" multiple> <option value="html">HTML</option> <option value="css">CSS</option> <option value="js">JavaScript</option> </select>

Datalist (автодоповнення)

<input list="cities" name="city"> <datalist id="cities"> <option value="Київ"> <option value="Львів"> <option value="Одеса"> </datalist>

💡 Кращі практики

✅ Рекомендації по вибору типів:
  • email/tel/url: Завжди замість text для відповідних даних
  • number: Тільки для чисел, які можна +/- (не ID, не телефон)
  • search: Для всіх пошукових полів
  • date/time: Замість text для дат, але з fallback для старих браузерів
  • hidden: Для технічних даних, ніколи для чутливої інформації
⚠️ Поширені помилки:
  • type="number" для телефонів (номер можна +/-, телефон - ні)
  • Відсутність labels (погана доступність)
  • Ігнорування autocomplete атрибутів
  • Placeholder замість label
  • type="text" замість спеціалізованих типів
  • Забування про мобільні клавіатури
