📏 Одиниці виміру

Повний посібник з абсолютних, відносних, viewport та функціональних одиниць

Типи одиниць виміру

CSS підтримує безліч одиниць виміру, які діляться на кілька категорій. Кожна має свої особливості та застосування:

📐

Абсолютні одиниці

Фіксовані розміри

  • px Пікселі (найпопулярніша)
  • pt Пункти (1pt = 1.33px)
  • pc Піки (1pc = 12pt)
  • in Дюйми (1in = 96px)
  • cm Сантиметри
  • mm Міліметри
🔗

Відносні одиниці

Залежать від контексту

  • % Відсоток від батьківського
  • em Відносно font-size елемента
  • rem Відносно root font-size
  • ex Висота символу 'x'
  • ch Ширина символу '0'
  • lh Висота рядка елемента
🖥️

Viewport одиниці

Відносно розміру екрану

  • vw 1% ширини viewport
  • vh 1% висоти viewport
  • vmin 1% меншої сторони
  • vmax 1% більшої сторони
  • vi Viewport inline (логічна)
  • vb Viewport block (логічна)
🧮

Функціональні одиниці

Обчислювані значення

  • calc() Математичні вирази
  • min() Найменше з значень
  • max() Найбільше з значень
  • clamp() Обмежене значення
  • fr Частка простору (Grid)

Абсолютні одиниці

Абсолютні одиниці мають фіксовані розміри, незалежно від інших елементів:

Одиниця Повна назва Еквівалент в px Використання Приклад
px Піксель 1px = 1px Веб-дизайн, границі, точне позиціонування width: 300px;
pt Пункт 1pt ≈ 1.33px Друк, документи font-size: 12pt;
pc Піка 1pc ≈ 16px Типографіка, друк margin: 1pc;
in Дюйм 1in = 96px Фізичні розміри, друк width: 2in;
cm Сантиметр 1cm ≈ 37.8px Фізичні розміри, друк height: 5cm;
mm Міліметр 1mm ≈ 3.78px Точні фізичні розміри border: 2mm solid;
💡 Коли використовувати абсолютні одиниці:
  • px - для границь, точного позиціонування, іконок
  • pt - для друкованих документів та email
  • cm/mm/in - для CSS друку та фізичних розмірів

Відносні одиниці

Відносні одиниці залежать від контексту - розміру шрифту, батьківського елемента або viewport:

em vs rem - важлива відмінність

CSS
/* em - відносно font-size ПОТОЧНОГО елемента */ .parent { font-size: 20px; } .child { font-size: 1.5em; /* 1.5 * 20px = 30px */ padding: 1em; /* 1 * 30px = 30px */ } /* rem - відносно font-size ROOT елемента (html) */ html { font-size: 16px; /* базовий розмір */ } .element { font-size: 1.5rem; /* 1.5 * 16px = 24px */ margin: 2rem; /* 2 * 16px = 32px */ }

Відсотки - залежать від властивості

CSS
.parent { width: 400px; height: 300px; font-size: 20px; } .child { width: 50%; /* 50% від width батьківського = 200px */ height: 75%; /* 75% від height батьківського = 225px */ font-size: 120%; /* 120% від font-size батьківського = 24px */ padding: 5%; /* 5% від WIDTH батьківського = 20px */ }
⚠️ Важливо: Відсотки для padding та margin завжди відраховуються від ширини батьківського елемента, навіть для вертикальних значень!

Viewport одиниці

Viewport одиниці дозволяють створювати елементи, розміри яких залежать від розміру екрану:

vw
1vw = 1% ширини екрану
10vw
vh
1vh = 1% висоти екрану
15vh
vmin
1% меншої сторони
8vmin
vmax
1% більшої сторони
6vmax
CSS
/* Повноекранні елементи */ .hero-section { width: 100vw; /* Повна ширина екрану */ height: 100vh; /* Повна висота екрану */ } /* Адаптивна типографіка */ .responsive-title { font-size: 4vw; /* Розмір залежить від ширини */ font-size: clamp(1.5rem, 4vw, 3rem); /* Обмежений діапазон */ } /* Квадратний елемент */ .square { width: 30vmin; /* Залежить від меншої сторони */ height: 30vmin; /* Завжди буде квадратом */ }
🎯 Популярні use cases для viewport одиниць:
  • 100vh - повноекранні секції (hero, splash screen)
  • vw - адаптивна типографіка і spacing
  • vmin/vmax - елементи з фіксованим співвідношенням сторін

Функціональні одиниці

CSS надає функції для складних обчислень та адаптивного дизайну:

calc() - математичні вирази

CSS
/* Змішування різних одиниць */ .container { width: calc(100% - 40px); /* Повна ширина мінус відступи */ height: calc(100vh - 200px); /* Повна висота мінус header */ margin: calc(1rem + 5px); /* Складне значення */ } /* Responsive grid */ .grid-item { width: calc(33.333% - 20px); /* 3 колонки з gap */ margin-right: 20px; } /* Центрування з offset */ .centered { position: absolute; left: calc(50% - 150px); /* Центр мінус половина ширини */ top: calc(50% - 100px); }

min(), max(), clamp() - адаптивні значення

CSS
/* min() - найменше значення */ .responsive-width { width: min(100%, 600px); /* Не більше 600px, але адаптивна */ } /* max() - найбільше значення */ .min-height { height: max(200px, 50vh); /* Мінімум 200px або 50% екрану */ } /* clamp() - значення в діапазоні */ .fluid-typography { font-size: clamp(1rem, 4vw, 2.5rem); /* мінімум 1rem, ідеально 4vw, максимум 2.5rem */ } .adaptive-padding { padding: clamp(1rem, 5%, 3rem); /* Адаптивні відступи з обмеженнями */ }

Практичні рекомендації

🎯 Коли використовувати кожну одиницю:
  • px - границі, тіні, точне позиціонування, іконки
  • rem - розміри шрифтів, відступи, розміри компонентів
  • em - відступи всередині компонентів (кнопки, input)
  • % - ширина контейнерів, адаптивний layout
  • vw/vh - повноекранні елементи, hero секції
  • vmin/vmax - квадратні елементи, медіа контент
⚠️ Поширені помилки:
  • Використання em для font-size може призвести до каскадного збільшення
  • Viewport одиниці можуть створювати проблеми на мобільних через address bar
  • Змішування px та відносних одиниць без розуміння контексту
  • Забування про accessibility при використанні viewport одиниць для тексту

Приклад сучасного підходу

CSS
/* CSS змінні для консистентності */ :root { --font-size-base: 1rem; --spacing-unit: 1rem; --content-width: min(1200px, 90vw); } /* Адаптивна типографіка */ .heading { font-size: clamp(1.5rem, 4vw, 3rem); line-height: 1.2; } /* Гнучкі контейнери */ .container { width: var(--content-width); margin: 0 auto; padding: calc(var(--spacing-unit) * 2); } /* Responsive spacing */ .section { padding-block: clamp(2rem, 8vh, 6rem); padding-inline: max(1rem, 5vw); }
