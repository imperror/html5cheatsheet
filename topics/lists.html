🔢 Типи нумерації для упорядкованих списків

Атрибут type дозволяє змінювати стиль нумерації

type="I"

  2. Пункт II
  3. Пункт III
Малі римські
type="i"
  1. Пункт i
  2. Пункт ii
  3. Пункт iii

📋 Пояснення атрибутів для упорядкованих списків:

  • start="число" - задає початкове значення нумерації
  • reversed - змінює порядок нумерації на зворотний
  • value="число" - задає конкретне значення для елемента li
  • type="1|A|a|I|i" - змінює тип нумерації (цифри, літери, римські)

🎮 Інтерактивна демонстрація атрибутів:

<ol start="5"> <li>П'ятий пункт</li> <li>Шостий пункт</li> <li>Сьомий пункт</li> </ol>
Результат:
  1. П'ятий пункт
  2. Шостий пункт
  3. Сьомий пункт

🎨 CSS властивості для стилізації списків:

  • list-style-type - тип маркера (disc, circle, square, none)
  • list-style-position - позиція маркера (inside, outside)
  • list-style-image - зображення як маркер
  • ::before псевдоелемент - кастомні маркери через CSS
  • counter() - власні лічильники для нумерації

✨ Кастомна стилізація списків:

<style> .custom-list-1 { list-style-type: none; padding-left: 0; } .custom-list-1 li::before { content: "🌟"; margin-right: 0.5rem; } </style> <ul class="custom-list-1"> <li>Елемент зі зірочкою</li> <li>Ще один елемент</li> <li>Третій елемент</li> </ul>
Результат:
  • Елемент зі зірочкою
  • Ще один елемент
  • Третій елемент

✅ Кращі практики

✅ Рекомендації:

  • Використовуйте <ol> коли порядок важливий (інструкції, етапи)
  • Використовуйте <ul> для звичайних переліків
  • Використовуйте <dl> для термінів та визначень
  • Завжди використовуйте <li> всередині <ul> або <ol>
  • Додавайте семантичний зміст через правильний вибір типу списку
  • Використовуйте CSS для стилізації замість HTML атрибутів

⚠️ Обережно:

  • Не зловживайте вкладеністю списків (максимум 3-4 рівні)
  • Уникайте порожніх <li> елементів
  • Не використовуйте списки тільки для відступів
  • Перевіряйте доступність для екранних читачів

❌ Уникайте:

  • Розміщення тексту напряму в <ul> або <ol> без <li>
  • Використання <br> замість списків для переліків
  • Змішування типів списків без логічної причини
  • Надмірної стилізації, яка псує семантику
