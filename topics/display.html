👁️ CSS Display та Visibility

Керування відображенням та видимістю елементів

🎯 Основи Display та Visibility

CSS надає потужні інструменти для керування тим, як елементи відображаються на сторінці:

  • display - визначає тип відображення елемента та його поведінку в потоці документа
  • visibility - керує видимістю елемента без впливу на макет

📦 Властивість Display

Властивість display є однією з найважливіших у CSS. Вона визначає, як елемент відображається та взаємодіє з іншими елементами.

display: block
div { display: block; }
Елемент займає всю доступну ширину та починається з нового рядка. За замовчуванням: div, p, h1-h6, header, section.
display: inline
span { display: inline; }
Елемент займає тільки необхідну ширину та не переноситься на новий рядок. За замовчуванням: span, a, strong, em.
display: inline-block
button { display: inline-block; }
Поєднує переваги inline та block: не переноситься на новий рядок, але можна задати розміри. За замовчуванням: button, input.
display: flex
container { display: flex; }
Створює flex-контейнер для гнучкого макетування дочірніх елементів. Дозволяє легко вирівнювати та розподіляти простір.
display: grid
container { display: grid; }
Створює grid-контейнер для двовимірного макетування. Найпотужніший інструмент для складних макетів.
display: none
element { display: none; }
Повністю приховує елемент та видаляє його з потоку документа. Елемент не займає місця в макеті.

👻 Властивість Visibility

Властивість visibility керує видимістю елемента, але залишає його в потоці документа:

visibility: visible
element { visibility: visible; }
Елемент видимий (значення за замовчуванням). Елемент займає своє місце в макеті та відображається нормально.
visibility: hidden
element { visibility: hidden; }
Елемент прихований, але займає місце в макеті. На відміну від display: none, простір залишається зарезервованим.
visibility: collapse
tr { visibility: collapse; }
Для таблиць: приховує рядок/колонку без впливу на макет. Для інших елементів працює як hidden.

💡 Практичні приклади використання

🎭 Модальні вікна та оверлеї

/* Приховування модального вікна */ .modal { display: none; /* Повністю приховано */ } .modal.active { display: flex; /* Показуємо як flex-контейнер */ position: fixed; top: 0; left: 0; width: 100%; height: 100%; }

🎨 Адаптивний дизайн

/* Приховування елементів на мобільних */ @media (max-width: 768px) { .desktop-only { display: none; } .mobile-nav { display: block; } }

🔄 Анімації та переходи

/* Анімація появи */ .fade-element { visibility: hidden; opacity: 0; transition: opacity 0.3s ease, visibility 0.3s ease; } .fade-element.visible { visibility: visible; opacity: 1; }
💡 Порада: Використовуйте visibility: hidden для анімацій, де потрібно зберегти місце елемента, та display: none для повного приховування без впливу на макет.

🎯 Найкращі практики

✅ Рекомендації:
  • Використовуйте display: flex або display: grid для сучасних макетів
  • Для анімацій переходів віддавайте перевагу visibility над display
  • Комбінуйте visibility з opacity для плавних ефектів
  • Використовуйте display: none для умовного відображення контенту
⚠️ Застереження:
  • Зміна display може викликати reflow - дорогу операцію
  • Елементи з visibility: hidden не доступні для клавіатурної навігації
  • Уникайте частих змін display в анімаціях
