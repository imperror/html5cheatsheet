🎯 Що таке веб-доступність?

Веб-доступність означає, що веб-сайти, інструменти та технології розроблені так, щоб люди з інвалідністю могли їх використовувати. Це включає людей з:

👁️ Порушення зору

Сліпота, слабкий зір, дальтонізм

  • Програми зчитування екрана
  • Збільшення тексту
  • Високий контрast
  • Голосове управління

👂 Порушення слуху

Глухота, зниження слуху

  • Субтитри та підписи
  • Мова жестів
  • Текстові альтернативи
  • Візуальні індикатори

🖐️ Фізичні обмеження

Обмеження рухливості рук

  • Навігація з клавіатури
  • Голосове управління
  • Переключення за допомогою
  • Адаптивні пристрої

🧠 Когнітивні особливості

Дислексія, СДВГ, аутизм

  • Простий інтерфейс
  • Чіткі інструкції
  • Контроль над часом
  • Уникнення миготіння

📋 WCAG 2.1 Guidelines

Web Content Accessibility Guidelines (WCAG) 2.1 — це міжнародний стандарт веб-доступності. Він базується на чотирьох принципах:

Рівень A

Мінімальний рівень доступності

  • Alt-текст для зображень
  • Навігація з клавіатури
  • Підписи для відео

Рівень AA

Стандартний рівень (рекомендований)

  • Коефіцієнт контрасту 4.5:1
  • Можливість зміни розміру тексту
  • Сумісність з допоміжними технологіями

Рівень AAA

Найвищий рівень доступності

  • Коефіцієнт контрасту 7:1
  • Мова жестів для аудіо
  • Допомога в читанні

🏷️ ARIA (Accessible Rich Internet Applications)

ARIA надає семантичну інформацію про елементи для допоміжних технологій:

/* Основні ARIA атрибути */ /* aria-label - Надає доступну назву елемента */ <button aria-label="Закрити вікно">✕</button> /* aria-labelledby - Посилається на інший елемент як на назву */ <h2 id="billing">Платіжна інформація</h2> <fieldset aria-labelledby="billing">...</fieldset> /* aria-describedby - Додатковий опис елемента */ <input type="password" aria-describedby="pwd-help"> <div id="pwd-help">Пароль має містити мінімум 8 символів</div> /* aria-expanded - Стан розкривного елемента */ <button aria-expanded="false" aria-controls="menu">Меню</button> <ul id="menu" aria-hidden="true">...</ul> /* aria-live - Оголошення динамічних змін */ <div aria-live="polite" id="status"></div> <div aria-live="assertive" id="errors"></div> /* role - Семантична роль елемента */ <div role="button" tabindex="0">Кастомна кнопка</div> <nav role="navigation">...</nav> <main role="main">...</main>

🎨 Контраст кольорів

Достатній контраст між текстом і фоном критично важливий для читабельності:

Перевірка контрасту

#000000
#ffffff
Приклад тексту для перевірки контрасту
Коефіцієнт контрасту: 21:1 ✅ AA ✅ AAA
💡 Стандарти контрасту:
  • WCAG AA: 4.5:1 для звичайного тексту, 3:1 для великого тексту
  • WCAG AAA: 7:1 для звичайного тексту, 4.5:1 для великого тексту
  • Великий текст: 18pt+ або 14pt+ жирний

⌨️ Навігація з клавіатури

Забезпечення можливості навігації всього сайту лише за допомогою клавіатури:

/* Навігація з клавіатури */ /* Tabindex для управління порядком фокусу */ <div tabindex="0">Фокусований елемент</div> <div tabindex="-1">Програмно фокусований</div> /* Стилізація фокусу */ button:focus { outline: 3px solid #4A90F4; outline-offset: 2px; } /* Прихована навігація для скрін-рідерів */ .skip-link { position: absolute; top: -40px; left: 6px; background: #000; color: #fff; padding: 8px; text-decoration: none; border-radius: 4px; } .skip-link:focus { top: 6px; } /* Обробка клавіш у JavaScript */ function handleKeyDown(event) { switch(event.key) { case 'Enter': case ' ': // Активація елемента event.preventDefault(); activateElement(); break; case 'Escape': // Закриття модального вікна closeModal(); break; case 'ArrowDown': // Навігація вниз focusNextItem(); break; } }
Демо навігації з клавіатури

Спробуйте навігацію клавішами Tab, Enter, та стрілками:

Фокусований div (Tab для фокусу, Enter для активації)
Натисніть Tab для переходу між елементами, Enter для активації

🔧 Семантичний HTML

Використання правильних HTML елементів для кращого розуміння допоміжними технологіями:

/* Семантичні елементи HTML5 */ /* Правильна структура заголовків */ <h1>Головний заголовок сторінки</h1> <h2>Розділ 1</h2> <h3>Підрозділ 1.1</h3> <h3>Підрозділ 1.2</h3> <h2>Розділ 2</h2> /* Семантичні елементи для структури */ <header> <nav aria-label="Головна навігація"> <ul> <li><a href="/">Головна</a></li> <li><a href="/about">Про нас</a></li> </ul> </nav> </header> <main> <article> <header> <h1>Заголовок статті</h1> <time datetime="2024-01-01">1 січня 2024</time> </header> <section> <h2>Вступ</h2> <p>Основний контент...</p> </section> </article> <aside> <h2>Пов'язані статті</h2> <nav aria-label="Пов'язані статті"> <ul>...</ul> </nav> </aside> </main> <footer> <p>© 2024 Моя компанія</p> </footer> /* Доступні форми */ <form> <fieldset> <legend>Особиста інформація</legend> <label for="name">Ім'я:</label> <input type="text" id="name" name="name" required aria-describedby="name-help"> <div id="name-help">Введіть ваше повне ім'я</div> <label for="email">Email:</label> <input type="email" id="email" name="email" required aria-invalid="false"> </fieldset> <button type="submit">Відправити</button> </form>

📱 Тестування доступності

Інструменти та методи для перевірки доступності вашого сайту:

🛠️ Автоматичні інструменти

  • axe DevTools
  • WAVE браузерне розширення
  • Lighthouse (Chrome DevTools)
  • Pa11y (командний рядок)
  • AccessibilityChecker.org

👥 Ручне тестування

  • Навігація лише з клавіатури
  • Тестування зі скрін-рідером
  • Перевірка контрасту
  • Тестування з вимкненим JavaScript
  • Масштабування до 200%

📱 Скрін-рідери

  • NVDA (Windows) - безкоштовний
  • JAWS (Windows) - платний
  • VoiceOver (Mac/iOS) - вбудований
  • TalkBack (Android) - вбудований
  • Orca (Linux) - безкоштовний

✅ Чеклист перевірки

  • Alt-текст для всіх зображень
  • Правильна структура заголовків
  • Достатній кольоровий контраст
  • Можливість навігації з клавіатури
  • Зрозумілі link та button тексти
⚠️ Важливо пам'ятати:
  • Автоматичні інструменти виявляють лише 20-30% проблем доступності
  • Найкращий спосіб тестування - залучення користувачів з інвалідністю
  • Доступність - це не одноразова задача, а постійний процес
