📸 Основи зображень в HTML

Тег <img> використовується для вставки зображень у веб-сторінки. Це самозакриваючий тег, який не потребує закриваючого тегу.

💡 Анатомія зображення

<img src="шлях_до_зображення" alt="опис_зображення">

src - обов'язковий атрибут з шляхом до файлу

alt - альтернативний текст для доступності

🎯 Основні атрибути зображень

Ключові атрибути для контролю відображення зображень

Атрибут Опис Приклад Примітки
src Шлях до файлу зображення src="images/photo.jpg" Обов'язковий атрибут
alt Альтернативний текст alt="Захід сонця над морем" Важливо для доступності
width Ширина зображення в пікселях width="300" Краще використовувати CSS
height Висота зображення в пікселях height="200" Краще використовувати CSS
title Підказка при наведенні title="Прекрасний захід" Спливаюча підказка
loading Контроль завантаження loading="lazy" lazy, eager
srcset Адаптивні зображення srcset="img-2x.jpg 2x" Для різних роздільностей
sizes Розміри для адаптивності sizes="(max-width: 768px) 100vw" Використовується з srcset

🎨 Формати зображень

Різні формати файлів та їх особливості

📷
JPEG / JPG
✅ Малий розмір файлу
✅ Підтримка мільйонів кольорів
❌ Втрата якості при стисненні
❌ Немає прозорості
Використання: фотографії, складні зображення з багатьма кольорами
🖼️
PNG
✅ Без втрати якості
✅ Підтримка прозорості
❌ Більший розмір файлу
❌ Не підходить для фото
Використання: логотипи, іконки, зображення з прозорістю
🎯
WebP
✅ Відмінне стиснення
✅ Підтримка прозорості
✅ Анімація
❌ Не всі браузери підтримують
Використання: сучасні веб-сайти з fallback
📐
SVG
✅ Векторна графіка
✅ Малий розмір
✅ Масштабується без втрат
❌ Не підходить для фото
Використання: іконки, логотипи, прості ілюстрації
🎬
GIF
✅ Підтримка анімації
✅ Широка підтримка
❌ Обмежена палітра (256 кольорів)
❌ Великий розмір для анімації
Використання: прості анімації, мемки
🆕
AVIF
✅ Найкраще стиснення
✅ Висока якість
✅ Підтримка HDR
❌ Обмежена підтримка браузерів
Використання: майбутнє веб-розробки

🖼️ Figure та Figcaption

Семантична розмітка для зображень з підписами

<figure> <img src="chart.png" alt="Графік продажів за 2024 рік"> <figcaption> Рис. 1: Динаміка продажів компанії за перше півріччя 2024 року </figcaption> </figure>

💡 Переваги Figure та Figcaption

<figure> - семантично групує зображення з підписом

<figcaption> - надає опис або підпис до зображення

SEO дружність - пошукові системи краще розуміють контекст

Доступність - екранні читачі правильно озвучують зв'язок

⚡ Оптимізація зображень

🚀 Поради щодо оптимізації:

  • Розмір файлу: стискайте зображення без втрати якості
  • Формат: обирайте правильний формат для типу зображення
  • Розміри: не завантажуйте зображення більші ніж потрібно
  • Lazy loading: використовуйте loading="lazy"
  • WebP: використовуйте сучасні формати з fallback
  • CDN: розміщуйте зображення на швидких серверах

♿ Доступність зображень

♿ Важливо для доступності:

  • alt атрибут: завжди додавайте описовий текст
  • Декоративні зображення: використовуйте alt=""
  • Інформативні зображення: опишіть зміст та контекст
  • Складні зображення: додавайте детальний опис
  • Текст у зображеннях: дублюйте в alt або на сторінці
  • Контраст: забезпечте достатній контраст

✅ Кращі практики

✅ Рекомендації:

  • Завжди додавайте описовий alt текст
  • Використовуйте правильний формат для типу зображення
  • Оптимізуйте розмір файлів без втрати якості
  • Використовуйте loading="lazy" для покращення продуктивності
  • Додавайте width та height для запобігання зсувам
  • Використовуйте srcset для адаптивних зображень

⚠️ Обережно:

  • Не забувайте про мобільні пристрої при виборі розмірів
  • Перевіряйте швидкість завантаження сторінки
  • Використовуйте fallback для нових форматів (WebP, AVIF)
  • Не покладайтесь тільки на зображення для передачі інформації
  • Тестуйте доступність з екранними читачами

❌ Уникайте:

  • Зображень без alt атрибуту
  • Використання зображень замість тексту
  • Завантаження занадто великих файлів
  • Неоптимізованих зображень для ретина-дисплеїв
  • Inline стилів для розмірів (використовуйте CSS)
  • Зображень, які блокують завантаження сторінки
