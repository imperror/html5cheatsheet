📊 Таблиці в HTML

Навчіться створювати структуровані таблиці для відображення табличних даних

📋 Основи HTML таблиць

Таблиці використовуються для відображення структурованих даних у рядках та стовпцях. Основні елементи: <table>, <tr>, <th> та <td>.

💡 Структура таблиці

<table> - контейнер таблиці

<tr> - рядок таблиці (table row)

<th> - заголовок стовпця (table header)

<td> - комірка з даними (table data)

Проста таблиця <table>
<table> <tr> <th>Ім'я</th> <th>Вік</th> <th>Місто</th> </tr> <tr> <td>Іван</td> <td>25</td> <td>Київ</td> </tr> <tr> <td>Марія</td> <td>30</td> <td>Львів</td> </tr> </table>
Ім'я Вік Місто
Іван 25 Київ
Марія 30 Львів

🏗️ Структурні елементи таблиць

Семантичні елементи для організації контенту таблиці

<thead>
Заголовок таблиці. Містить рядки з заголовками стовпців. Зазвичай стилізується окремо від основного контенту.
<tbody>
Тіло таблиці. Містить основні дані таблиці. Може бути кілька tbody елементів для групування рядків.
<tfoot>
Підвал таблиці. Містить підсумкові дані, такі як суми, середні значення тощо.
<caption>
Заголовок таблиці. Описує зміст таблиці та покращує доступність для екранних читачів.
<colgroup>
Група стовпців. Дозволяє застосовувати стилі до цілих стовпців або груп стовпців.
<col>
Окремий стовпець. Використовується всередині colgroup для визначення властивостей стовпців.

♿ Доступність таблиць

♿ Важливо для доступності:

  • scope атрибут: вказує зв'язок між заголовками та комірками
  • caption елемент: описує зміст таблиці
  • headers атрибут: явно зв'язує комірки з заголовками
  • summary атрибут: короткий опис структури таблиці
  • Логічна послідовність: заголовки мають йти перед даними
  • Контраст: достатній контраст між текстом та фоном

⚡ Продуктивність та оптимізація

🚀 Поради щодо продуктивності:

  • table-layout: fixed; прискорює рендеринг великих таблиць
  • Віртуалізація: показуйте тільки видимі рядки для великих даних
  • Пагінація: розбивайте великі таблиці на сторінки
  • Lazy loading: завантажуйте дані по мірі потреби
  • Кешування: зберігайте відсортовані дані
  • Мінімізація DOM: уникайте зайвих елементів

✅ Кращі практики

✅ Рекомендації:

  • Використовуйте семантичні елементи (thead, tbody, tfoot)
  • Додавайте scope атрибути для заголовків
  • Включайте caption для опису таблиці
  • Забезпечуйте адаптивність для мобільних пристроїв
  • Використовуйте CSS для стилізації замість HTML атрибутів
  • Додавайте hover ефекти для кращого UX

⚠️ Обережно:

  • Не використовуйте таблиці для макета сторінки
  • Тестуйте доступність з екранними читачами
  • Врахуйте продуктивність для великих таблиць
  • Забезпечуйте достатній контраст кольорів
  • Уникайте надмірного використання colspan/rowspan

❌ Уникайте:

  • Таблиць без заголовків або caption
  • Використання таблиць для позиціонування
  • Фіксованих розмірів без адаптивності
  • Надмірно складних структур
  • Пропуску scope атрибутів для складних таблиць
  • Inline стилів замість CSS класів
