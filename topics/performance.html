⚡ Продуктивність веб-сайтів

Оптимізація швидкості завантаження та користувацького досвіду

🎯 Чому продуктивність важлива?

Швидкість завантаження сайту критично впливає на користувацький досвід, SEO рейтинг та конверсію. Дослідження показують:

👥 Користувацький досвід

Вплив на поведінку відвідувачів

  • 53% користувачів залишають сайт, якщо він завантажується > 3 сек
  • Затримка в 1 сек зменшує задоволення на 16%
  • Швидкі сайти мають вищий рейтинг задоволення

🔍 SEO рейтинг

Вплив на позиції в пошуку

  • Google використовує швидкість як фактор ранжування
  • Core Web Vitals впливають на позиції
  • Мобільна швидкість особливо важлива

💰 Бізнес-показники

Вплив на доходи та конверсію

  • Amazon: +1% швидкості = +1% доходів
  • Walmart: -1 сек = +2% конверсія
  • Pinterest: -40% час завантаження = +15% реєстрацій

📱 Мобільні пристрої

Особливості мобільної продуктивності

  • Повільніші процесори та з'єднання
  • Обмежена пропускна спроможність
  • Більше залежність від швидкості

📊 Core Web Vitals

Ключові метрики користувацького досвіду, які Google використовує для ранжування:

🏃‍♂️

Largest Contentful Paint (LCP)

Час завантаження найбільшого елемента

< 2.5 сек
Вимірює швидкість завантаження
📱

First Input Delay (FID)

Затримка першої взаємодії

< 100 мс
Вимірює інтерактивність
🎯

Cumulative Layout Shift (CLS)

Стабільність візуального контенту

< 0.1
Вимірює візуальну стабільність
⚡

First Contentful Paint (FCP)

Час появи першого контенту

< 1.8 сек
Вимірює сприйняття швидкості

🖼️ Оптимізація зображень

Зображення часто складають 60-70% розміру сторінки. Їх оптимізація дає найбільший ефект:

Сучасні формати зображень:

WebP
-25-30% менше JPEG
✅ Рекомендовано
AVIF
-50% менше JPEG
✅ Найкраще
JPEG
Базовий розмір
⚠️ Застарілий
PNG
+40-60% більше
❌ Тільки для альфа

Сучасні формати з fallback */ <picture> <source srcset="image.avif" type="image/avif"> <source srcset="image.webp" type="image/webp"> <img src="image.jpg" alt="Опис зображення" loading="lazy"> </picture> /* 2. Адаптивні зображення */ <img srcset="image-320w.webp 320w, image-640w.webp 640w, image-1280w.webp 1280w" sizes="(max-width: 320px) 280px, (max-width: 640px) 600px, 1200px" src="image-640w.webp" alt="Адаптивне зображення" loading="lazy"> /* 3. CSS оптимізація зображень */ img { max-width: 100%; height: auto; /* Плавне з'явлення */ opacity: 0; transition: opacity 0.3s; } img.loaded { opacity: 1; } /* 4. Lazy loading з Intersection Observer */ const imageObserver = new IntersectionObserver((entries, observer) => { entries.forEach(entry => { if (entry.isIntersecting) { const img = entry.target; img.src = img.dataset.src; img.classList.add('loaded'); imageObserver.unobserve(img); } }); }); document.querySelectorAll('img[data-src]').forEach(img => { imageObserver.observe(img); });

Зменшення розміру CSS, JavaScript та HTML файлів:

/* Мініфікація та стиснення */ /* 1. HTML мініфікація */ <!-- До мініфікації --> <div class="container"> <h1>Заголовок</h1> <p>Параграф тексту</p> </div> <!-- Після мініфікації --> <div class="container"><h1>Заголовок</h1><p>Параграф тексту</p></div> /* 2. CSS мініфікація */ /* До */ .container { width: 100%; max-width: 1200px; margin: 0 auto; padding: 2rem; } /* Після */ .container{width:100%;max-width:1200px;margin:0 auto;padding:2rem} /* 3. Gzip стиснення в .htaccess */ <IfModule mod_deflate.c> AddOutputFilterByType DEFLATE text/html AddOutputFilterByType DEFLATE text/css AddOutputFilterByType DEFLATE text/javascript AddOutputFilterByType DEFLATE application/javascript AddOutputFilterByType DEFLATE application/json </IfModule> /* 4. Brotli стиснення (кращий за Gzip) */ <IfModule mod_brotli.c> AddOutputFilterByType BROTLI_COMPRESS text/html AddOutputFilterByType BROTLI_COMPRESS text/css AddOutputFilterByType BROTLI_COMPRESS application/javascript AddOutputFilterByType BROTLI_COMPRESS application/json </IfModule> /* 5. Critical CSS inline */ <style> /* Критичний CSS для above-the-fold контенту */ body{font-family:Arial,sans-serif;margin:0} .header{background:#333;color:#fff;padding:1rem} </style> /* 6. Відкладене завантаження CSS */ <link rel="preload" href="styles.css" as="style" onload="this.onload=null;this.rel='stylesheet'"> <noscript><link rel="stylesheet" href="styles.css"></noscript>

💾 Кешування

Правильне налаштування кешування значно покращує швидкість повторних візитів:

/* Стратегії кешування */ /* 1. HTTP заголовки кешування в .htaccess */ <IfModule mod_expires.c> ExpiresActive on # Зображення (1 рік) ExpiresByType image/jpg "access plus 1 year" ExpiresByType image/jpeg "access plus 1 year" ExpiresByType image/png "access plus 1 year" ExpiresByType image/webp "access plus 1 year" ExpiresByType image/svg+xml "access plus 1 year" # CSS та JavaScript (1 місяць) ExpiresByType text/css "access plus 1 month" ExpiresByType application/javascript "access plus 1 month" # HTML (1 година) ExpiresByType text/html "access plus 1 hour" # Fonts (1 рік) ExpiresByType font/woff2 "access plus 1 year" ExpiresByType font/woff "access plus 1 year" </IfModule> /* 2. Service Worker для кешування */ // Реєстрація Service Worker if ('serviceWorker' in navigator) { navigator.serviceWorker.register('/sw.js'); } // sw.js - Service Worker файл const CACHE_NAME = 'site-cache-v1'; const urlsToCache = [ '/', '/styles.css', '/script.js', '/images/logo.png' ]; // Встановлення SW та кешування self.addEventListener('install', event => { event.waitUntil( caches.open(CACHE_NAME) .then(cache => cache.addAll(urlsToCache)) ); }); // Обробка запитів self.addEventListener('fetch', event => { event.respondWith( caches.match(event.request) .then(response => { // Повертаємо кешовану версію або завантажуємо з мережі return response || fetch(event.request); }) ); }); /* 3. Browser caching з правильними заголовками */ // Cache-Control заголовки Cache-Control: public, max-age=31536000 // 1 рік для статичних ресурсів Cache-Control: public, max-age=3600 // 1 година для HTML Cache-Control: no-cache // Завжди перевіряти актуальність /* 4. CDN кешування */ // Використання CDN для статичних ресурсів <link href="https://cdn.example.com/css/styles.css" rel="stylesheet"> <script src="https://cdn.example.com/js/app.js"></script>

🚀 Техніки оптимізації

Найефективніші методи покращення продуктивності:

📦 Code Splitting

Розділення JavaScript на менші модулі, які завантажуються за потребою

Покращення: 30-50%

🔄 Tree Shaking

Видалення невикористаного коду з JavaScript бандлів

Зменшення розміру: 20-40%

📱 Resource Hints

Попереднє завантаження критичних ресурсів

Покращення LCP: 10-30%

⚡ Critical CSS

Інлайн CSS для above-the-fold контенту

Покращення FCP: 15-25%

🖼️ Image Optimization

WebP/AVIF формати, адаптивні розміри, lazy loading

Зменшення розміру: 50-70%

📡 HTTP/2 Push

Проактивна відправка критичних ресурсів

Покращення швидкості: 10-20%
