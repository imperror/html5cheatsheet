🔺 SVG Векторна Графіка

Масштабована векторна графіка для веб

🎯 Основи SVG

SVG (Scalable Vector Graphics) - це формат векторної графіки, що базується на XML:

/* Базовий синтаксис SVG */ <svg width="200" height="150" xmlns="http://www.w3.org/2000/svg"> <!-- Прямокутник --> <rect x="10" y="10" width="80" height="60" fill="#667eea" stroke="#333" stroke-width="2"/> <!-- Коло --> <circle cx="150" cy="40" r="30" fill="#e53e3e" opacity="0.8"/> <!-- Текст --> <text x="10" y="130" font-family="Arial" font-size="18" fill="#333"> SVG Graphics </text> </svg>
Приклад базових SVG фігур

Прямокутник, коло, еліпс, трикутник, лінія та текст

📐 Основні SVG елементи

Найпоширеніші елементи для створення SVG графіки:

Базові фігури

/* Основні фігури SVG */ <!-- Прямокутник --> <rect x="10" y="10" width="100" height="80" fill="blue" stroke="black" stroke-width="2" rx="5" ry="5"/> <!-- Коло --> <circle cx="100" cy="100" r="50" fill="red" stroke="green" stroke-width="3"/> <!-- Еліпс --> <ellipse cx="150" cy="80" rx="60" ry="40" fill="yellow" opacity="0.5"/> <!-- Лінія --> <line x1="0" y1="0" x2="100" y2="100" stroke="purple" stroke-width="4"/> <!-- Ламана лінія --> <polyline points="0,40 40,40 40,80 80,80 80,120" fill="none" stroke="orange" stroke-width="3"/> <!-- Багатокутник --> <polygon points="50,5 100,35 82,85 18,85 0,35" fill="cyan" stroke="navy" stroke-width="2"/>

Шляхи (Paths)

/* Складні фігури за допомогою path */ <!-- Прості команди --> <path d="M 10 10 L 100 10 L 100 100 Z" fill="lightblue" stroke="blue"/> <!-- Криві Безьє --> <path d="M 20 20 Q 50 5 80 20 T 140 20" fill="none" stroke="red" stroke-width="3"/> <!-- Дуги --> <path d="M 50 50 A 25 25 0 1 1 100 50" fill="none" stroke="green" stroke-width="4"/> /* Команди path: M/m - Move to (переміститись) L/l - Line to (лінія до) H/h - Horizontal line (горизонтальна лінія) V/v - Vertical line (вертикальна лінія) C/c - Cubic Bezier curve (кубічна крива) Q/q - Quadratic Bezier curve (квадратична крива) A/a - Arc (дуга) Z/z - Close path (закрити шлях) */

🎨 Стилізація та властивості

SVG підтримує багато способів стилізації елементів:

Кольори та заповнення

/* Способи задання кольорів */ <!-- Назви кольорів --> <rect fill="red" stroke="blue"/> <!-- HEX значення --> <circle fill="#667eea" stroke="#333333"/> <!-- RGB значення --> <ellipse fill="rgb(102, 126, 234)" stroke="rgb(51, 51, 51)"/> <!-- Прозорість --> <rect fill="red" opacity="0.5"/> <circle fill="blue" fill-opacity="0.3" stroke-opacity="0.8"/> <!-- Градієнти --> <defs> <linearGradient id="grad1" x1="0%" y1="0%" x2="100%" y2="0%"> <stop offset="0%" style="stop-color:#667eea;stop-opacity:1"/> <stop offset="100%" style="stop-color:#764ba2;stop-opacity:1"/> </linearGradient> </defs> <rect fill="url(#grad1)"/>

Обводка та контури

/* Властивості stroke */ <!-- Основні властивості --> <line stroke="black" stroke-width="5"/> <!-- Стиль лінії --> <line stroke-dasharray="5,5"/> <!-- Пунктирна --> <line stroke-dasharray="10,5,2,5"/> <!-- Складна --> <!-- Закінчення ліній --> <line stroke-linecap="round"/> <!-- round, square, butt --> <!-- З'єднання ліній --> <polyline stroke-linejoin="round"/> <!-- round, miter, bevel --> <!-- Зміщення пунктиру --> <line stroke-dashoffset="3"/>

🎬 SVG Анімації

SVG підтримує різні види анімацій:

SMIL анімації

/* SMIL анімації (вбудовані в SVG) */ <!-- Анімація руху --> <circle cx="50" cy="50" r="10" fill="red"> <animateMotion dur="3s" repeatCount="indefinite"> <path d="M 50 50 Q 100 25 150 50 T 250 50"/> </animateMotion> </circle> <!-- Анімація атрибутів --> <rect x="10" y="10" width="50" height="50" fill="blue"> <animate attributeName="width" from="50" to="150" dur="2s" repeatCount="indefinite"/> <animate attributeName="fill" values="blue;red;green;blue" dur="3s" repeatCount="indefinite"/> </rect> <!-- Анімація трансформацій --> <g transform-origin="50 50"> <animateTransform attributeName="transform" type="rotate" values="0;360" dur="2s" repeatCount="indefinite"/> <rect x="25" y="25" width="50" height="50" fill="green"/> </g>

CSS анімації для SVG

/* CSS анімації */ <style> .rotating-element { animation: rotate 2s linear infinite; transform-origin: center; } @keyframes rotate { from { transform: rotate(0deg); } to { transform: rotate(360deg); } } .pulsing-element { animation: pulse 1.5s ease-in-out infinite alternate; } @keyframes pulse { from { transform: scale(1); opacity: 1; } to { transform: scale(1.2); opacity: 0.7; } } .moving-element { animation: move 3s ease-in-out infinite; } @keyframes move { 0%, 100% { transform: translateX(0); } 50% { transform: translateX(100px); } } </style> <svg width="300" height="200"> <circle cx="50" cy="50" r="20" fill="red" class="rotating-element"/> <rect x="100" y="30" width="40" height="40" fill="blue" class="pulsing-element"/> <ellipse cx="200" cy="50" rx="25" ry="15" fill="green" class="moving-element"/> </svg>

📊 SVG vs Canvas vs CSS

Порівняння різних технологій для графіки у веб:

Характеристика SVG Canvas CSS
Тип графіки Векторна Растрова (піксельна) Векторна (обмежена)
Масштабування ✓ Без втрат якості ✗ Втрата якості ✓ Без втрат якості
DOM інтеграція ✓ Повна інтеграція ✗ Елемент без структури ✓ Повна інтеграція
Продуктивність ⚠ Повільніше при багатьох елементах ✓ Висока при складній графіці ✓ Дуже швидка
Інтерактивність ✓ Події на кожен елемент ✗ Тільки на canvas ✓ Події на кожен елемент
Анімації ✓ CSS + SMIL ✓ JavaScript (60 FPS) ✓ CSS transitions/animations
SEO ✓ Індексується ✗ Не індексується ✓ Індексується
Доступність ✓ Screen readers ✗ Потребує додаткової роботи ✓ Screen readers
Складність створення ✓ Декларативна ⚠ Імперативна (JS) ✓ Декларативна
🎯 Коли використовувати що:
  • SVG: Іконки, логотипи, прості діаграми, інфографіка
  • Canvas: Ігри, складні візуалізації, обробка зображень
  • CSS: Прості ефекти, переходи, базові фігури

♿ Доступність та кращі практики

✅ Кращі практики SVG:
  • Використовуйте title та desc для опису контенту
  • Додавайте role="img" для декоративних SVG
  • Оптимізуйте код - видаляйте зайве
  • Використовуйте viewBox для адаптивності
  • Групуйте пов'язані елементи в <g>
  • Виносьте стилі в CSS замість inline атрибутів
/* Приклад доступного SVG */ <svg role="img" aria-labelledby="chart-title" aria-describedby="chart-desc" viewBox="0 0 400 300" xmlns="http://www.w3.org/2000/svg"> <title id="chart-title">Продажі за 2024 рік</title> <desc id="chart-desc"> Стовпчиковий графік показує зростання продажів з 1000 одиниць у січні до 5000 одиниць у грудні. </desc> <g role="img" aria-label="Січень: 1000 продажів"> <rect x="50" y="200" width="30" height="50" fill="#667eea"/> <text x="65" y="270" text-anchor="middle">Січ</text> </g> <!-- Інші стовпці... --> </svg> /* CSS для кращої доступності */ svg { max-width: 100%; height: auto; } /* Прихована альтернатива для screen readers */ .sr-only { position: absolute; width: 1px; height: 1px; padding: 0; margin: -1px; overflow: hidden; clip: rect(0,0,0,0); white-space: nowrap; border: 0; }
⚠️ Поширені помилки:
  • Відсутність title/desc для складного контенту
  • Використання тільки кольору для передачі інформації
  • Надто складні SVG (потрібно оптимізувати)
  • Відсутність fallback для старих браузерів
  • Ігнорування responsive поведінки
