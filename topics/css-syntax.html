⚡ Синтаксис CSS та Селектори

Детальний розгляд структури CSS правил та способів вибору елементів

Анатомія CSS правила

Кожне CSS правило складається з декількох частин. Розберемо їх детально:

селектор {
  властивість: значення;
  інша-властивість: інше-значення;
}
  • Селектор - вказує, які елементи стилізувати
  • Властивість - що змінювати (колір, розмір, тощо)
  • Значення - на що змінювати
  • Пунктуація - фігурні дужки та крапка з комою
CSS
/* Приклад повного CSS правила */ h1 { color: #2d3748; /* Колір тексту */ font-size: 2rem; /* Розмір шрифту */ text-align: center; /* Вирівнювання */ margin-bottom: 1rem; /* Відступ знизу */ } /* Правило для параграфів */ p { line-height: 1.6; /* Висота рядка */ color: #4a5568; /* Сірий колір */ }
💡 Важливо: Кожне оголошення властивості повинно закінчуватися крапкою з комою ;. Остання властивість у блоці може не мати крапку з комою, але краще завжди її ставити.

Коментарі в CSS

Коментарі допомагають пояснювати код і тимчасово відключати стилі:

CSS
/* Це однорядковий коментар */ /* Це багаторядковий коментар Може займати декілька рядків Корисно для детальних пояснень */ h1 { color: blue; /* Коментар наприкінці рядка */ /* font-size: 3rem; - цей стиль відключений */ text-align: center; } /* ======================================== РОЗДІЛ: Стилі для навігації ======================================== */
⚠️ Увага: У CSS коментарі завжди пишуться у форматі /* коментар */. Одинарні слеші // НЕ працюють!

Типи селекторів CSS

Селектори визначають, до яких HTML елементів застосовуються стилі. Існує багато типів селекторів:

🏷️ Селектор тегу

h1 { color: red; } p { font-size: 16px; }

Вибирає всі елементи вказаного тегу

🎯 Селектор класу

.header { background: blue; } .button { padding: 10px; }

Вибирає елементи з атрибутом class

🆔 Селектор ID

#main { width: 100%; } #sidebar { float: left; }

Вибирає елемент з унікальним id

⭐ Універсальний селектор

* { margin: 0; padding: 0; }

Вибирає всі елементи на сторінці

📋 Селектор атрибуту

[type="text"] { border: 1px solid; } a[href^="https"] { color: green; }

Вибирає елементи за атрибутами

🔗 Комбіновані селектори

div p { color: blue; } .header > h1 { font-size: 2rem; }

Комбінують декілька селекторів

Детальніше про селектори

Селектори нащадків та дітей

CSS
/* Селектор нащадків (пробіл) */ div p { color: blue; /* Всі p всередині div */ } /* Селектор прямих дітей (>) */ div > p { color: red; /* Тільки p - прямі діти div */ } /* Сусідній селектор (+) */ h1 + p { font-weight: bold; /* p одразу після h1 */ } /* Загальний сусідній селектор (~) */ h1 ~ p { margin-left: 20px; /* Всі p після h1 на тому ж рівні */ }

Псевдокласи

CSS
/* Стани посилань */ a:link { color: blue; } /* Не відвідані */ a:visited { color: purple; } /* Відвідані */ a:hover { color: red; } /* При наведенні */ a:active { color: orange; } /* При кліку */ /* Структурні псевдокласи */ p:first-child { font-weight: bold; } p:last-child { margin-bottom: 0; } p:nth-child(2n) { background: #f0f0f0; } /* Парні */ /* Псевдокласи форм */ input:focus { border-color: blue; } input:disabled { opacity: 0.5; }

Псевдоелементи

CSS
/* Псевдоелементи створюють віртуальні елементи */ p::before { content: "👉 "; /* Додає контент перед p */ color: blue; } p::after { content: " ✓"; /* Додає контент після p */ color: green; } p::first-line { font-weight: bold; /* Стилізує перший рядок */ } p::first-letter { font-size: 2em; /* Велика перша літера */ float: left; }

Специфічність (Specificity)

Коли до одного елемента застосовуються декілька CSS правил, браузер визначає пріоритет за специфічністю:

Тип селектора Вага Приклад Специфічність
Inline стилі 1000 style="color: red" 1,0,0,0
ID 100 #header 0,1,0,0
Класи, атрибути, псевдокласи 10 .nav, [type], :hover 0,0,1,0
Теги, псевдоелементи 1 div, ::before 0,0,0,1
Приклади специфічності
/* Специфічність: 0,0,0,1 */ p { color: black; } /* Специфічність: 0,0,1,0 */ .text { color: blue; } /* Специфічність: 0,1,0,0 */ #main { color: green; } /* Специфічність: 0,1,1,1 */ #main .text p { color: red; } /* !important перемагає все (не рекомендується) */ p { color: purple !important; }
🎯 Правило специфічності: Чим вища специфічність, тим вищий пріоритет. При однаковій специфічності виграє останнє правило у коді.

Каскадність CSS

Каскадність - це механізм, який визначає, які стилі застосувати, коли є конфлікт. Пріоритет (від найвищого):

Джерела стилів
  1. Inline стилі (style="...")
  2. Внутрішні стилі (<style>)
  3. Зовнішні стилі (<link>)
  4. Стилі браузера (за замовчуванням)
Фактори пріоритету
  1. !important
  2. Специфічність
  3. Порядок у коді
  4. Наслідування
Приклад каскадності
<!-- HTML --> <p class="highlight" id="intro" style="color: purple;"> Текст з кількома стилями </p> <!-- CSS --> <style> p { color: black; } /* Специфічність: 0,0,0,1 */ .highlight { color: blue; } /* Специфічність: 0,0,1,0 */ #intro { color: green; } /* Специфічність: 0,1,0,0 */ </style> /* Результат: текст буде фіолетовим через inline стиль */

Практичні поради

🎯 Кращі практики селекторів:
  • Використовуйте класи для повторюваних стилів
  • ID тільки для унікальних елементів
  • Уникайте надто довгих ланцюжків селекторів
  • Називайте класи семантично (.header, .button)
  • Мінімізуйте використання !important
⚠️ Поширені помилки:
  • Забування крапки з комою після властивостей
  • Неправильні назви властивостей (color-text замість color)
  • Зловживання !important
  • Використання тегових селекторів для стилізації
