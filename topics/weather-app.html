Покроковий туторіал створення сучасного веб-додатку для перегляду прогнозу погоди з використанням HTML, CSS та JavaScript

В цьому міні-проєкті ми створимо функціональний додаток прогнозу погоди, який дозволить:

🌍 Пошук міст

Введення назви міста та отримання актуальної інформації про погоду

💡 Підхід до розробки: Ми почнемо з чистого HTML та CSS, створимо макет з тестовими даними, а потім додамо JavaScript для інтерактивності та API інтеграцію.

1Створюємо HTML файл

Спочатку створимо базовий HTML файл. Збережіть його як index.html на вашому комп'ютері.

🔧 Що робимо:

Створюємо HTML5 документ з правильною структурою та підключенням CSS файлу.

<!DOCTYPE html> <html lang="uk"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Прогноз погоди</title> <link rel="stylesheet" href="style.css"> </head> <body> <div class="container"> <h1>🌤️ Прогноз погоди</h1> <p>Тут буде наш додаток!</p> </div> </body> </html>

📖 Пояснення коду:

  • <!DOCTYPE html> - каже браузеру, що це HTML5 документ
  • <html lang="uk"> - вказує українську мову для доступності
  • <meta viewport> - налаштування для мобільних пристроїв
  • <link rel="stylesheet"> - підключаємо CSS файл для стилів
  • <div class="container"> - основний контейнер для всього вмісту
  • 🌤️ - емодзі як іконка (краще ніж картинки для простих проектів)

2Створюємо header з формою пошуку

Додамо семантичний заголовок з полем пошуку та кнопками. Використаємо HTML5 семантичні теги для кращої структури.

🔧 Що додаємо:

Замініть вміст всередині <div class="container"> на цей код:

<div class="container"> <header class="header"> <h1>🌤️ Прогноз погоди</h1> <div class="search-container"> <input type="text" id="cityInput" placeholder="Введіть назву міста..."> <button id="searchBtn">🔍 Пошук</button> <button id="locationBtn">📍 Моє місце</button> </div> </header> </div>

📖 Нові HTML5 елементи:

  • <header> - семантичний тег для заголовка сторінки
  • class="header" - CSS клас для стилізації (будемо використовувати пізніше)
  • <input type="text"> - поле для введення тексту
  • id="cityInput" - унікальний ідентифікатор для JavaScript
  • placeholder - підказка що вводити (зникає при введенні)
  • div class="search-container" - контейнер для групування кнопок
  • 📍 Моє місце - кнопка для геолокації (додамо пізніше)

3Створюємо main секцію з поточною погодою

Додамо семантичний <main> з детальною структурою для відображення поточної погоди.

🔧 Що додаємо:

Додайте після </header> всередині container:

<main class="weather-content"> <div class="current-weather"> <div class="weather-icon"> <span id="currentIcon">☀️</span> </div> <div class="weather-info"> <h2 id="cityName">Київ</h2> <div class="temperature"> <span id="currentTemp">22</span><span class="unit">°C</span> </div> <div class="weather-description" id="weatherDesc">Сонячно</div> </div> </div> </main>

📖 Семантична структура:

  • <main> - головний контент сторінки (семантичний HTML5 тег)
  • class="weather-content" - CSS клас для стилізації всього weather контенту
  • class="current-weather" - контейнер для поточної погоди
  • class="weather-icon" - окремий блок для іконки погоди
  • class="weather-info" - блок з текстовою інформацією
  • class="temperature" - спеціальний контейнер для температури
  • class="unit" - окремий клас для "°C" (інша стилізація)

4Додаємо детальну інформацію про погоду

Створимо блок з додатковою інформацією: відчувається як, вологість, вітер, тиск.

🔧 Що додаємо:

Додайте після </div> (закриття current-weather) всередині main:

<div class="weather-details"> <div class="detail-item"> <span class="label">Відчувається як:</span> <span id="feelsLike">25°C</span> </div> <div class="detail-item"> <span class="label">Вологість:</span> <span id="humidity">60%</span> </div> <div class="detail-item"> <span class="label">Вітер:</span> <span id="windSpeed">5 м/с</span> </div> <div class="detail-item"> <span class="label">Тиск:</span> <span id="pressure">1013 гПа</span> </div> </div>

📖 Нова структура:

  • class="weather-details" - контейнер для всіх деталей
  • class="detail-item" - кожен параметр в окремому блоці
  • class="label" - текст підпису (Вологість:, Вітер:)
  • id - унікальні ідентифікатори для зміни через JavaScript
  • Одиниці виміру: °C, %, м/с, гПа (гектопаскаль)

5Додаємо прогноз на тиждень

Створимо секцію з прогнозом погоди на 7 днів. Поки що залишимо порожнім - дані додамо через JavaScript.

🔧 Що додаємо:

Додайте після </div> (закриття weather-details) всередині main:

<div class="forecast"> <h3>📅 Прогноз на 7 днів</h3> <div class="forecast-container" id="forecastContainer"> <!-- Дані будуть додані JavaScript --> </div> </div>

📖 Структура прогнозу:

  • class="forecast" - загальний контейнер для секції прогнозу
  • <h3> - заголовок секції (менший ніж h2)
  • class="forecast-container" - контейнер для карточок днів
  • id="forecastContainer" - для JavaScript додавання контенту
  • HTML коментар - пояснення що тут буде

💡 Чому порожній? Прогноз буде динамічно створюватися через JavaScript для гнучкості.

6Закриваємо main та підключаємо JavaScript

Завершимо HTML структуру та підготуємо підключення JavaScript файлу.

🔧 Що додаємо:

Закрийте main тег та додайте підключення script перед </body>:

</main> </div> <script src="script.js"></script> </body> </html>

📖 Завершення HTML структури:

  • </main> - закриваємо головний контент
  • </div> - закриваємо container
  • <script src="script.js"> - підключаємо JavaScript файл
  • Порядок важливий: script завжди перед закриттям body

🎯 Результат: Повна семантична HTML структура готова! Тепер переходимо до стилізації.

13Реальний API та UX покращення

Замінимо тестові дані реальним API прогнозу погоди та додамо покращення користувацького досвіду.

🔑 API Ключ: Зареєструйтеся на openweathermap.org та отримайте безкоштовний API ключ.
⏰ Примітка: API ключ може активуватися 10 хвилин - 2 години після реєстрації.

⚠️ ВАЖЛИВО: Безпека API ключів

Зазвичай API ключі НІКОЛИ не розміщують у frontend коді! Будь-хто може переглянути ваш код у браузері та скопіювати ключ.

🌤️ OpenWeatherMap - виключення: Можна використовувати в frontend, оскільки:

  • ✅ Безкоштовні ліміти: 1000 запитів/день (у гіршому випадку хтось "вкраде" ваші запити про погоду)
  • ✅ Без фінансових ризиків: Немає автоматичних списань коштів
  • ✅ Обмеження домену: Можна налаштувати дозволені домени

💳 Небезпечні API (НЕ для frontend):

  • ❌ Платіжні системи: Stripe, PayPal - можуть списати кошти
  • ❌ Cloud сервіси: AWS, Google Cloud - дорогі послуги
  • ❌ SMS/Email API: Можуть відправляти спам від вашого імені
  • ❌ AI API: OpenAI, зазвичай дорогі за використання

🛡️ Для production: Створіть власний backend API, який буде проксувати запити до зовнішніх сервісів і приховувати справжні ключі.

🌐 API інтеграція та UX покращення:

// Функція показу завантаження function showLoading() { const weatherContent = document.querySelector('.weather-content'); weatherContent.classList.add('loading'); cityName.textContent = 'Завантаження...'; currentTemp.textContent = '--'; weatherDesc.textContent = 'Отримання даних...'; currentIcon.textContent = '⏳'; } function hideLoading() { const weatherContent = document.querySelector('.weather-content'); weatherContent.classList.remove('loading'); } // Функція отримання погоди з API async function fetchWeatherData(city) { try { showLoading(); // Поточна погода const currentResponse = await fetch( `${API_BASE_URL}/weather?q=${encodeURIComponent(city)}&appid=${API_KEY}&units=metric&lang=uk` ); if (!currentResponse.ok) { if (currentResponse.status === 404) { throw new Error('Місто не знайдено'); } else if (currentResponse.status === 401) { throw new Error('Невірний API ключ'); } else { throw new Error('Помилка сервера'); } } const currentData = await currentResponse.json(); // Прогноз на 5 днів (кожні 3 години) const forecastResponse = await fetch( `${API_BASE_URL}/forecast?q=${encodeURIComponent(city)}&appid=${API_KEY}&units=metric&lang=uk` ); const forecastData = await forecastResponse.json(); // Обробка та відображення даних const weatherData = processWeatherData(currentData, forecastData); updateWeather(weatherData); saveLastCity(city.toLowerCase()); showMessage(`Погода оновлена для: ${weatherData.city}`, 'success'); } catch (error) { hideLoading(); console.error('Weather API Error:', error); // Fallback на тестові дані const mockCity = city.toLowerCase(); if (mockData[mockCity]) { updateWeather(mockData[mockCity]); showMessage(`API недоступний. Показано тестові дані для ${mockData[mockCity].city}`, 'info'); } else { showMessage(`Помилка: ${error.message}`, 'error'); } } } // Функція обробки API даних function processWeatherData(current, forecast) { const weatherIcons = { '01d': '☀️', '01n': '🌙', '02d': '⛅', '02n': '⛅', '03d': '☁️', '03n': '☁️', '04d': '☁️', '04n': '☁️', '09d': '🌧️', '09n': '🌧️', '10d': '🌦️', '10n': '🌦️', '11d': '⛈️', '11n': '⛈️', '13d': '❄️', '13n': '❄️', '50d': '🌫️', '50n': '🌫️' }; const days = ['Нд', 'Пн', 'Вт', 'Ср', 'Чт', 'Пт', 'Сб']; // Сьогодні const processedForecast = [{ day: 'Сьогодні', icon: weatherIcons[current.weather[0].icon] || '🌤️', high: Math.round(current.main.temp_max), low: Math.round(current.main.temp_min) }]; // Групування прогнозу по днях const dailyForecasts = {}; forecast.list.forEach(item => { const date = new Date(item.dt * 1000); const dateKey = date.toDateString(); if (!dailyForecasts[dateKey]) { dailyForecasts[dateKey] = { temps: [], icons: [], date: date }; } dailyForecasts[dateKey].temps.push(item.main.temp); dailyForecasts[dateKey].icons.push(item.weather[0].icon); }); // Наступні 6 днів const sortedDates = Object.keys(dailyForecasts).sort(); sortedDates.slice(0, 6).forEach((dateKey, index) => { const dayData = dailyForecasts[dateKey]; const date = dayData.date; const dayName = index === 0 ? 'Завтра' : days[date.getDay()]; // Найпопулярніша іконка const iconCounts = {}; dayData.icons.forEach(icon => { iconCounts[icon] = (iconCounts[icon] || 0) + 1; }); const mostFrequentIcon = Object.keys(iconCounts).reduce((a, b) => iconCounts[a] > iconCounts[b] ? a : b ); processedForecast.push({ day: dayName, icon: weatherIcons[mostFrequentIcon] || '🌤️', high: Math.round(Math.max(...dayData.temps)), low: Math.round(Math.min(...dayData.temps)) }); }); return { city: current.name, temperature: Math.round(current.main.temp), description: current.weather[0].description, icon: weatherIcons[current.weather[0].icon] || '🌤️', feelsLike: Math.round(current.main.feels_like), humidity: current.main.humidity, windSpeed: Math.round(current.wind ? current.wind.speed : 0), pressure: current.main.pressure, forecast: processedForecast }; } // Оновлена функція пошуку function searchCity() { const city = cityInput.value.trim(); if (!city) { showMessage('Введіть назву міста!', 'error'); return; } // Якщо API ключ налаштований, використовуємо API if (API_KEY && API_KEY !== '') { fetchWeatherData(city); } else { // Використовуємо тестові дані const mockCity = city.toLowerCase(); if (mockData[mockCity]) { updateWeather(mockData[mockCity]); saveLastCity(mockCity); showMessage(`Показано тестові дані для ${mockData[mockCity].city}`, 'info'); } else { showMessage('Місто не знайдено! Спробуйте: Київ, Львів, Одеса', 'error'); } } cityInput.value = ''; } // Оновлена геолокація з API function getCurrentLocationWeather() { if (!navigator.geolocation) { showMessage('Геолокація не підтримується вашим браузером', 'error'); return; } showLoading(); showMessage('Визначення вашого місцезнаходження...', 'info'); navigator.geolocation.getCurrentPosition( async (position) => { const { latitude, longitude } = position.coords; if (API_KEY && API_KEY !== '') { try { const response = await fetch( `${API_BASE_URL}/weather?lat=${latitude}&lon=${longitude}&appid=${API_KEY}&units=metric&lang=uk` ); const data = await response.json(); cityInput.value = data.name; fetchWeatherData(data.name); } catch (error) { hideLoading(); showMessage('Не вдалося отримати дані для вашого місцезнаходження', 'error'); } } else { hideLoading(); updateWeather(mockData['київ']); showMessage('Показано дані для Києва (API недоступний)', 'info'); } }, (error) => { hideLoading(); let errorMessage = 'Помилка геолокації'; switch(error.code) { case error.PERMISSION_DENIED: errorMessage = 'Доступ до геолокації заборонено'; break; case error.POSITION_UNAVAILABLE: errorMessage = 'Інформація про місцезнаходження недоступна'; break; case error.TIMEOUT: errorMessage = 'Час очікування геолокації вичерпано'; break; } showMessage(errorMessage, 'error'); } ); }
